# 医院手术排班系统使用说明（详细版）

这份文档面向一线使用人员（管理员、医生）和部署维护人员，按“从安装到日常使用”的顺序编写。

> 文档导航：[`README.md`](./README.md) | [`DOCTOR_QUICK_GUIDE.md`](./DOCTOR_QUICK_GUIDE.md) | [`ADMIN_OPERATION_GUIDE.md`](./ADMIN_OPERATION_GUIDE.md)

---

## 1. 系统简介

医院手术排班系统用于管理手术安排，核心能力包括：

- 手术排班的新增、编辑、删除、查询
- 手术室时间冲突自动校验
- 管理员与医生分角色权限
- 数据库备份、恢复与统计
- Web 访问（桌面/手机浏览器）

---

## 2. 角色与权限

系统有两个角色：`admin`（管理员）和 `doctor`（医生）。

### 2.1 管理员（admin）

可执行：

- 查看全部手术
- 创建/编辑/删除任意手术
- 创建/修改/删除医生账号
- 修改手术状态（已安排/已完成/已取消）
- 使用数据库管理功能（统计、备份、恢复、清空）

### 2.2 医生（doctor）

可执行：

- 查看自己创建的手术
- 创建自己的手术
- 编辑/删除自己的手术
- 查看指定条件下的手术信息（受权限控制）

不可执行：

- 用户管理
- 数据库管理
- 操作其他医生的手术（除管理员开放的全量视图场景）

## 3. 安装与启动

## 3.1 环境要求

- Node.js `>= 14`
- npm `>= 6`

建议使用较新的 Node LTS 版本（18/20）。

## 3.2 安装依赖

在项目根目录执行：

```bash
npm install
cd client
npm install
cd ..
```

或使用一键命令：

```bash
npm run install-all
```

## 3.3 配置环境变量

在根目录创建或编辑 `.env`：

```env
PORT=3000
JWT_SECRET=your-secret-key-change-in-production
NODE_ENV=development
```

说明：

- `PORT`：后端端口
- `JWT_SECRET`：登录令牌签名密钥（生产环境必须修改）
- `NODE_ENV`：运行环境

## 3.4 启动开发模式

```bash
npm run dev
```

系统会同时启动：

- 后端：`http://localhost:3000`
- 前端：`http://localhost:3001`（开发模式）

## 3.5 默认账号

- 用户名：`admin`
- 密码：`admin123`

首次进入后建议立即修改默认管理密码（可通过用户管理替换管理员账号策略，或在数据库层面执行密码更新）。

---

## 4. 登录与首页

1. 打开登录页
2. 输入用户名和密码
3. 点击“登录”
4. 成功后进入对应角色首页

如果登录失败，常见原因：

- 用户名或密码错误
- 数据库损坏或连接异常
- 后端服务未启动

---

## 5. 管理员日常操作

## 5.1 新增医生账号

路径：`管理员页面 -> 用户管理`

步骤：

1. 点击“用户管理”
2. 新增用户，填写：
   - 用户名（唯一）
   - 密码
   - 姓名
   - 角色（doctor）
3. 保存后即可用于医生登录

注意：

- 用户名不能重复
- 不建议使用过弱密码

## 5.2 新增手术

路径：`管理员页面 -> 新增手术`

必填字段：

- 医生
- 病人姓名
- 手术项目
- 手术日期
- 手术室
- 开始时间
- 结束时间

可选字段：

- 状态（默认已安排）
- 备注

系统行为：

- 自动检查同一手术室同一日期的时间冲突
- 冲突时会提示无法保存

## 5.3 编辑手术

路径：列表行内“编辑”按钮

可修改：

- 医生（管理员可改）
- 患者、手术项目、日期、手术室、时间
- 状态、备注

注意：

- 若改了“手术室/日期/时间”，会重新做冲突校验

## 5.4 删除手术

路径：列表行内“删除”按钮

执行后将永久删除该条手术记录，请谨慎操作。

## 5.5 手术室时间可视化

在手术表单中填写“手术室 + 手术日期”后，点击“查看时间表”可查看占用情况：

- 用于快速避开冲突时段
- 可在编辑场景辅助调整时间

## 5.6 数据库管理

路径：`管理员页面 -> 数据库管理`

功能说明：

- 统计：查看用户数、手术数、完成数、数据库大小
- 备份：导出 JSON 备份文件
- 恢复：导入备份（合并/替换模式）
- 清空：危险操作，仅保留 admin

建议：

- 每天至少备份一次
- 重大操作前先备份

---

## 6. 医生日常操作

## 6.1 查看我的手术

登录后默认可查看个人相关排班，支持按日期筛选。

## 6.2 新增手术

医生创建手术时，医生字段默认绑定当前账号，不可越权创建其他医生名下手术。

## 6.3 编辑/删除手术

医生只能编辑和删除自己创建的手术。

---

## 7. 关键业务规则（必须了解）

- 同一手术室、同一天，时间区间不能重叠
- 手术结束时间应晚于开始时间
- 用户名全局唯一
- 权限校验在后端执行，前端仅作展示控制

---

## 8. 备份与恢复规范（推荐流程）

## 8.1 日常备份

建议频率：每天 1 次，或每次排班大量变更后立即备份。

操作：

1. 管理员进入数据库管理
2. 导出备份文件并妥善保存（建议含日期命名）

## 8.2 恢复策略

- `merge`（合并）：保留现有数据，导入不重复数据（推荐常规使用）
- `replace`（替换）：先清空再导入（仅用于完整恢复）

执行前建议：

1. 先导出当前备份
2. 在低峰时段操作
3. 恢复后抽样检查关键日期排班

## 10. 常见问题与排查

## 10.1 启动时报 sqlite3 架构错误

典型提示：`incompatible architecture`

处理：

```bash
npm run rebuild-sqlite3
```

必要时删除 `node_modules` 后重装依赖。

## 10.2 端口被占用（3000/3001）

表现：启动报 `EADDRINUSE`

处理：

1. 结束占用进程
2. 或修改端口后重启

## 10.3 登录后接口 401/403

检查：

- token 是否失效
- 是否使用了错误角色账号
- 后端 `JWT_SECRET` 是否在重启后变化导致 token 失效

## 10.4 看不到手术或数据异常

检查：

- 日期筛选是否正确
- 当前账号权限是否允许查看
- 数据是否被误删除或刚恢复

---

## 11. 操作安全建议

- 生产环境必须修改 `JWT_SECRET`
- 禁止长期使用默认管理员密码
- 定期备份数据库文件与 JSON 备份
- 限制服务器访问范围（内网优先）
- 对重要操作建立审批流程（如清库、替换恢复）

---

## 12. 推荐培训流程（给医院科室）

建议按以下顺序培训新用户：

1. 医生登录与个人排班增删改
2. 冲突规则与时间可视化使用
3. 管理员用户管理与跨医生排班
4. 备份/恢复演练
5. 异常场景（密码错误、冲突、权限不足）演示
