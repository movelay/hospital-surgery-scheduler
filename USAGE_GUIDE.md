## 1. 系统概述

### 系统能做什么

- 管理每天各手术室的手术安排（新增 / 编辑 / 删除 / 查询）
- 自动检测手术室时间冲突，避免排班撞车
- 管理员与医生分角色使用，权限分明
- 支持数据备份、恢复、统计
- 电脑和手机都能用

### 两种角色

| 能力 | 管理员（admin） | 医生（doctor） |
|------|:-:|:-:|
| 查看所有手术 | ✅ | ❌（仅自己） |
| 新增/编辑/删除手术 | 全部 | 仅自己 |
| 修改手术状态 | ✅ | ❌ |
| 用户管理 | ✅ | ❌ |
| 数据库管理 | ✅ | ❌ |

## 2. 安装与启动

### 环境要求

- **Node.js** >= 14（建议 18 或 20 LTS 版本）
- **npm** >= 6

### 安装步骤

```bash
# 一键安装所有依赖（后端 + 前端）
npm run install-all

# 构建前端
npm run build

# 启动生产服务
NODE_ENV=production npm start
```

启动成功后，终端会显示：

```
默认管理员账户已创建: admin / admin123
服务器运行在端口 3000
本地访问: http://localhost:3000
内网访问: http://192.168.x.x:3000
```

### 开发模式（调试用）

```bash
npm run dev
```

同时启动后端（3000 端口）和前端热重载（3001 端口）。

### 配置项

编辑项目根目录的 `.env` 文件：

```env
PORT=3000                                        # 服务端口
JWT_SECRET=your-secret-key-change-in-production   # JWT 签名密钥（生产必改）
NODE_ENV=production                               # 运行环境
```

## 3. 登录系统

打开浏览器访问服务地址，进入登录页面。

![登录页面](docs/screenshots/01-login.png)

**操作步骤：**

1. 输入用户名和密码
2. 点击「登录」按钮
3. 管理员自动进入管理面板，医生进入医生面板

**默认管理员账户：**

| 项目 | 值 |
|------|------|
| 用户名 | `admin` |
| 密码 | `admin123` |

> 首次使用务必修改默认密码。

**登录失败常见原因：**

- 用户名或密码错误
- 后端服务未启动
- 网络不通（手机端需确保与服务器同一网络）

## 4. 管理员操作

管理员登录后进入管理面板，顶部有三个快捷入口：**用户管理**、**数据库管理**、**退出登录**。

![管理员面板](docs/screenshots/02-admin-dashboard.png)

### 4.1 手术排班管理

#### 查看手术

- 默认显示当天所有手术
- 通过顶部**日期选择器**切换查看不同日期
- 每条手术显示：日期、时间、手术室、医生、病人、项目、状态

#### 新增手术

点击绿色的「新增手术」按钮，弹出手术表单：

![新增手术表单](docs/screenshots/03-surgery-form.png)

**必填字段：**

| 字段 | 说明 |
|------|------|
| 医生 | 从下拉列表选择（管理员专属） |
| 病人姓名 | 患者姓名 |
| 手术项目 | 例如：阑尾切除术 |
| 手术日期 | 选择日期 |
| 手术室 | 例如：1号手术室 |
| 开始时间 | 手术开始时间 |
| 结束时间 | 手术结束时间（必须晚于开始时间） |

**可选字段：** 状态（默认「已安排」）、备注

**冲突检测：** 保存时系统自动检查同一手术室同一天的时间是否有重叠，冲突时会提示无法保存。

#### 编辑手术

点击手术卡片上的「编辑」按钮，修改信息后保存。修改手术室、日期或时间时会重新校验冲突。

#### 删除手术

点击「删除」按钮，确认后永久删除。删除后不可撤销，请谨慎操作。

#### 手术状态

管理员可将手术状态修改为：

- **已安排**（scheduled）：默认状态
- **已完成**（completed）：手术已完成
- **已取消**（cancelled）：手术取消

---

### 4.2 用户管理

点击顶部「用户管理」按钮，打开用户管理面板。

![用户管理](docs/screenshots/04-user-management.png)

#### 新增用户

1. 点击「新增用户」
2. 填写用户名（唯一）、密码、姓名、角色（doctor）
3. 保存

#### 编辑用户

点击用户卡片上的「编辑」，可修改姓名、用户名、角色。如填写新密码则同时更新密码。

#### 删除用户

点击「删除」。若该用户有关联手术记录，系统会提示先处理手术后再删除。

> **建议**：使用规范命名（如拼音或工号）方便管理。

---

### 4.3 数据库管理

点击顶部「数据库管理」按钮，打开数据管理面板。

![数据库管理](docs/screenshots/05-database-management.png)

#### 数据统计

顶部卡片展示：

- **用户数量**：系统中的账号总数
- **手术记录**：手术总条数
- **已完成手术**：状态为「已完成」的数量
- **数据库大小**：SQLite 数据库文件大小

#### 数据备份

提供两种备份：

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| 导出备份（不含密码） | 仅导出业务数据 | 日常归档、数据分析 |
| 导出完整备份 | 包含用户密码哈希 | 完整数据迁移、灾难恢复 |

点击后自动下载 JSON 备份文件，文件名包含时间戳。

#### 数据恢复

1. 点击「选择备份文件」，选择之前导出的 JSON 文件
2. 选择恢复模式：
   - **合并导入**（推荐）：保留现有数据，仅导入不重复的记录
   - **替换导入**：清空后完整导入
3. 确认执行

> **重要**：恢复前请先导出当前数据备份。

#### 清空数据

高危操作，会删除所有用户和手术数据，仅保留 admin 账户。

## 5. 医生操作

医生使用管理员创建的账号登录后，进入医生工作面板。

![医生面板](docs/screenshots/06-doctor-dashboard.png)

### 查看手术

- 默认显示「我的手术」
- 通过日期选择器切换日期
- 可切换到「手术室视图」查看某日全部手术室的安排

### 新增手术

1. 点击「新增手术」
2. 医生字段自动绑定为当前登录账号（不可更改）
3. 填写病人、项目、日期、手术室、时间
4. 保存

### 编辑 / 删除手术

医生只能编辑和删除**自己创建**的手术。操作方式与管理员相同。

### 避免时间冲突的技巧

1. 先选好手术室和日期
2. 点击「手术室视图」查看该手术室当天的占用时段
3. 选择空闲时段再填写开始/结束时间

## 6. 多平台使用

### 电脑浏览器

直接在浏览器中访问 `http://服务器IP:3000`，支持所有主流浏览器（Chrome、Firefox、Safari、Edge）。

### 手机浏览器

手机连接到同一 WiFi 后，在手机浏览器中输入同样的地址。系统已做响应式适配：

- 表格自动转为卡片布局
- 表单变为底部弹出式
- 按钮和输入框适配触屏操作

### 手机 PWA（推荐）

将网页添加到主屏幕后，可像原生 App 一样使用：

**iOS（Safari）：**
1. 打开系统地址
2. 点击底部 **分享按钮**（方框+箭头图标）
3. 选择「**添加到主屏幕**」
4. 点击「添加」

**Android（Chrome）：**
1. 打开系统地址
2. 点击右上角 **三点菜单**
3. 选择「**添加到主屏幕**」或「**安装应用**」
4. 确认安装

### 桌面应用（Electron）

```bash
# 打包 Mac 版
npm run electron:build

# 打包 Windows 版
npm run electron:build:win
```

安装包生成在 `dist` 目录，双击即可安装使用。

## 7. 生产部署

### 内网部署（推荐）

适用于医院内部网络，所有设备通过局域网访问。

```bash
# 1. 安装依赖
npm run install-all

# 2. 构建前端
npm run build

# 3. 配置环境变量
# 编辑 .env，修改 JWT_SECRET

# 4. 启动服务
NODE_ENV=production npm start
```

局域网内其他电脑/手机通过 `http://服务器IP:3000` 访问。

### 后台运行（推荐）

使用 `pm2` 进程管理器保持服务持续运行：

```bash
# 安装 pm2
npm install -g pm2

# 启动服务
NODE_ENV=production pm2 start server/index.js --name hospital-scheduler

# 查看状态
pm2 status

# 设为开机自启
pm2 startup
pm2 save
```

### 外网部署

如需通过互联网访问，需要：

1. 购买云服务器（阿里云 / 腾讯云等）
2. 部署步骤与内网相同
3. 配置域名和 HTTPS（强烈建议）
4. 使用 Nginx 反向代理：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 8. 数据备份与恢复

### 日常备份策略

| 频率 | 操作 |
|------|------|
| 每天 | 导出一次备份文件 |
| 大量数据变更后 | 立即追加备份 |
| 执行高危操作前 | 先导出完整备份 |

### 备份操作

1. 管理员登录 → 点击「数据库管理」
2. 点击「导出备份」或「导出完整备份」
3. 将下载的 JSON 文件妥善保存（建议按日期命名）

### 恢复操作

1. 管理员登录 → 数据库管理 → 数据恢复
2. 选择备份文件
3. 选择恢复模式（合并 / 替换）
4. 确认执行

### 手动备份

也可以直接复制数据库文件：

```bash
cp data/hospital.db data/hospital_backup_$(date +%Y%m%d).db
```

## 9. 故障排查

### 服务启动失败

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| `EADDRINUSE` | 端口被占用 | 终止占用进程或修改 `.env` 中的 `PORT` |
| `incompatible architecture` | sqlite3 架构不匹配 | 执行 `npm run rebuild-sqlite3` |
| 启动无响应 | 依赖未安装 | 执行 `npm run install-all` |

### 登录问题

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 登录失败 | 账号密码错误 | 核对账号；联系管理员重置 |
| 401/403 错误 | Token 过期或 JWT_SECRET 变更 | 退出重新登录 |
| 页面空白 | 前端未构建 | 执行 `npm run build` 后重启 |

### 数据问题

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 看不到手术 | 日期筛选不对 | 检查日期选择器 |
| 数据丢失 | 执行了清空/替换恢复 | 从最近的备份文件恢复 |
| 手术冲突报错 | 时间段重叠 | 换时间或换手术室 |

### 完全重置

删除数据库文件后重启，系统会自动创建全新数据库：

```bash
rm data/hospital.db
NODE_ENV=production npm start
```

## 10. 安全建议

| 项目 | 建议 |
|------|------|
| JWT_SECRET | 生产环境必须修改为高强度随机字符串 |
| 管理员密码 | 首次登录后立即修改默认密码 |
| 网络访问 | 优先内网部署；外网必须配 HTTPS |
| 数据备份 | 每天至少备份一次 |
| 账号管理 | 定期清理离职人员账号 |
| 敏感操作 | 清空数据、替换恢复前先备份 |
