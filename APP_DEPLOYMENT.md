## 🚀 部署步骤

### 1. 服务端部署

1. **安装服务端 App**
   ```bash
   # 双击 DMG 文件，将 App 拖拽到 Applications 文件夹
   ```

2. **启动服务端**
   - 打开"医院手术排班系统-服务端"应用
   - 应用会自动启动 Express 服务器
   - 服务器状态窗口会显示运行日志
   - 默认监听端口: **3000**

3. **配置服务器地址**
   - 服务端 App 启动后会自动显示内网访问地址
   - 在服务端状态窗口中可以看到:
     - 本地访问: `http://localhost:3000`
     - 内网访问: `http://服务器IP:3000` (例如: `http://192.168.1.100:3000`)
   - 内网中的其他设备可以通过内网访问地址访问服务

4. **自动重启功能**
   - 如果端口 3000 已被占用，服务端会自动检测并关闭旧进程
   - 然后重新启动服务器，确保服务正常运行

5. **内网访问**
   - 服务器启动后会自动监听所有网络接口 (0.0.0.0)
   - 内网中的设备可以通过 `http://服务器IP:3000` 访问
   - 支持浏览器和客户端 App 访问

6. **数据库位置**
   - 数据库文件存储在: `~/Library/Application Support/医院手术排班系统-服务端/hospital.db`
   - 建议定期备份此文件

### 2. 客户端部署

1. **安装客户端 App**
   ```bash
   # 在每个医生的电脑上双击 DMG 文件安装
   ```

2. **首次运行配置**
   - 打开"医院手术排班系统"应用
   - 如果无法连接到服务器，会显示连接失败页面
   - 在连接失败页面输入服务器地址:
     - 本地服务器: `http://localhost:3000`
     - 远程服务器: `http://服务器IP:3000` 或 `http://服务器域名:3000`
   - 点击"保存设置并连接"

3. **使用应用**
   - 登录账号（默认管理员: `admin` / `admin123`）
   - 开始使用手术排班功能

## 🔧 网络配置

### 局域网部署

如果服务器和客户端在同一局域网:

1. **服务器端**:
   - 确保防火墙允许端口 3000
   - 获取服务器 IP 地址: `ifconfig` 或 `ipconfig`

2. **客户端**:
   - 使用服务器 IP 地址连接，例如: `http://192.168.1.100:3000`

### 远程部署

如果需要跨网络访问:

1. **服务器端**:
   - 配置路由器端口转发（端口 3000）
   - 或使用内网穿透工具（如 ngrok、frp）
   - 确保有公网 IP 或域名

2. **客户端**:
   - 使用公网地址或域名连接

## 📝 注意事项

1. **代码签名**
   - 当前 App 未进行代码签名
   - 首次打开时，macOS 可能会提示"无法验证开发者"
   - 解决方法: 系统设置 → 安全性与隐私 → 点击"仍要打开"

2. **防火墙**
   - 确保服务器防火墙允许端口 3000
   - macOS 防火墙设置: 系统设置 → 网络 → 防火墙

3. **数据库备份**
   - 定期备份数据库文件
   - 可以使用应用内的"数据库管理"功能进行备份

4. **性能优化**
   - 建议服务器配置: 至少 2GB RAM
   - 客户端配置: 普通办公电脑即可

## 🐛 故障排查

### 客户端无法连接服务器

1. **检查服务器是否运行**
   - 确认服务端 App 已启动
   - 检查服务器状态窗口

2. **检查网络连接**
   - 测试能否访问: `http://服务器地址:3000`
   - 在浏览器中打开测试

3. **检查防火墙**
   - 确保端口 3000 未被阻止

4. **检查服务器地址**
   - 确认输入的地址格式正确
   - 包含 `http://` 前缀

### 服务器启动失败

1. **检查端口占用**
   ```bash
   lsof -i :3000
   ```

2. **检查数据库权限**
   - 确保应用有写入用户数据目录的权限

3. **查看日志**
   - 检查服务器状态窗口的错误信息
